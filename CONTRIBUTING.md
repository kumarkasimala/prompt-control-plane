# Contributing

Contributions are welcome when they improve the clarity, portability, or usefulness of the prompt runtime architecture.

## Useful Contributions

- Schema improvements for prompt runtime objects.
- Examples for RAG, tool use, structured outputs, or guardrails.
- Evaluation cases for prompt regression testing.
- Telemetry mappings for OpenTelemetry-style traces.
- Documentation that makes the architecture easier to adopt.

## Guidelines

- Keep examples vendor-neutral and public.
- Do not include private customer data, secrets, proprietary prompts, or confidential implementation details.
- Prefer small pull requests with clear motivation.
- Include an example or test when changing schemas or evaluation files.
